### 1. **Paradigmas de Linguagens de Programação**

#### **A. Imperativas**

* **Ideia:** descrevem *como* o computador deve realizar tarefas.
* **Características:** controle de fluxo (loops, condições), alterações de estado (variáveis mutáveis).
* **Exemplos:** C, Java, Python (quando usado imperativamente), Fortran.
* **Subtipos:**

  * **Procedural:** organiza código em funções ou procedimentos (ex.: C).
  * **Orientada a Objetos:** organiza código em objetos que interagem (ex.: Java, C#).

#### **B. Declarativas**

* **Ideia:** descrevem *o que* deve ser feito, sem detalhar os passos.
* **Características:** foco no resultado, não no controle do fluxo.
* **Exemplos e subcategorias:**

  1. **Funcional:** funções puras, composição, imutabilidade (Haskell, Lisp).
  2. **Lógica:** fatos e regras; sistema deduz respostas (Prolog).
  3. **Consultas/Query:** descrevem dados a recuperar (SQL).
  4. **Markup/Descritivas:** descrevem estrutura/apresentação (HTML, CSS).
  5. **Regras/Constraints:** definem restrições; motor resolve (CLP).
  6. **Configuração/Infra:** definem estado desejado de sistemas (Terraform, Nix).

#### **C. Outras categorização (menos tradicionais)**

* **Concorrente/Paralela:** foca em tarefas simultâneas (Go, Erlang).
* **Reativa:** foca em mudanças de estado ou eventos ao longo do tempo (RxJS).
* **Orientada a Fluxos (Dataflow):** define fluxo de dados e transformações (LabVIEW).

Isso significa que se uma IA como ChatGPT fosse uma linguagem, ela seria declarativa. Claro que ela tende a trazer resultados probabilísticos, o que é diferente das linguagens, que são determinísticas. Por mais que um SQL mude a forma de consultar, os valor dos dados a serem retornados são os mesmos, já numa IA é por probalidadade.
  Resumindo: 
    SQL = declarativo determinístico. 
    ChatGPT = declarativo probabilístico.
